Workflow Building Tips
{# Tip 1: Dynamic Variables #}

Use #name to make variables dynamic

When you want a parameter to be adjustable across multiple workflow runs, use # followed by the parameter name in place of the actual value.

Example:
Instead of: temperature = 300
Use: temperature = #temperature
This makes "temperature" a dynamic parameter that can be varied in parameter sweeps or optimization runs.
{# Tip 2: Repeat vs Dynamic #}

Repeat is for static workflows only

  • Static (Repeat): Runs the exact same workflow multiple times with identical parameters
  • Dynamic (#name): Automatically triggers the Parameter Sweep option, allowing you to vary parameters across runs
Use repeat when you want statistical averaging of identical runs. Use dynamic parameters (e.g., #temperature) when you want to explore different parameter values.
{# Tip 3: Parameter Sweep #}

Automatically enabled when using dynamic parameters

Parameter sweep allows you to systematically explore different combinations of parameter values.

Triggered by: Using #name syntax (e.g., #temperature) in your workflow

Use cases: Sensitivity analysis, parameter optimization, design space exploration

{# Tip 4: Optimizer #}

Enabled when you have both dynamic parameters AND output values

The optimizer automatically finds the best parameter values to optimize your output metric.

Requirements:
  • At least one dynamic parameter using #name syntax (input)
  • At least one output value with "Save value as" name (objective function)

Note: Parameter sweep option remains available even in optimizer mode

{# Tip 5: Output Values #}

Use "Save value as" to capture important results

When you want to track or optimize a result from your workflow, give it a meaningful name in the "Save value as" field.

Naming conventions:
• Use descriptive names: final_stress, total_energy, efficiency
• Use underscores instead of spaces: max_temperature not max temperature
• Keep names concise but clear

Saved values become available for optimization and can be tracked across parameter sweeps.

{# Tip 7: Best Practices #}

  • Name parameters clearly: Use descriptive names like #mesh_size instead of #x
  • Set reasonable ranges: Define min/max values that make physical sense for your problem
  • Start simple: Begin with 1-2 parameters before adding more complexity
  • Document outputs: Clearly label which outputs are optimization objectives
  • Test first: Run a single workflow before launching sweeps or optimizations
  • Use batch mode wisely: Mark setup steps as "once per batch" to avoid unnecessary repetition
